5 · Risk Management — Simultaneous Multi-Hub Drop Model
⚡ Trigger Thresholds
Level 1 — Watch:
· Single hub: >15% impression drop in 7 days
· Alert sent to dashboard, no pause

Level 2 — Alert:
· Single hub: >20% drop in 7 days
· Publish queue paused for that hub
· Operator notified immediately

Level 3 — Critical:
· 2+ hubs: >20% drop simultaneously
· All hubs paused
· Full audit initiated
🛡️ Isolation Protocol
Why hubs are isolated:
· Separate ASN = different network origin
· Separate DNS = no shared NS pattern
· Separate IP = no IP-block correlation
· Separate theme/plugins = no CMS fingerprint
· Different publish days = no temporal pattern

Result:
A simultaneous drop affecting 2+ hubs indicates an algorithm update (sitewide signal), not a network pattern detection. Isolation means one hub cannot "contaminate" another.
🔄 Recovery Actions
Immediate (0–24h):
· Auto-pause affected hub queue
· Revert last 5 articles to draft
· GSC manual inspection

Short-term (1–7 days):
· Content quality audit
· Anchor distribution review
· Template rotation refresh

Recovery (7–30 days):
· Gradual resume (1 post/week)
· Monitor re-indexation
· Resume normal cadence if stable
6 · Rollback Flow
GSC Signal
>20% impression drop across ≥3 articles from same hub within 7 days
→
Auto-Pause
Hub publish queue paused. No new articles scheduled from this hub.
→
Revert Draft
Last 5 published articles set to draft + noindex via WP REST API.
→
Alert Sent
Metabase alert + email notification to operator with full log.
→
Audit
Content quality, anchor profile, and template pattern review.
→
Gradual Resume
1 post/week cadence after 7 stable days. Normal resume after 30 days.
